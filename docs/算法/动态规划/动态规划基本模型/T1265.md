# 🏆 算法大师题解：最长公共子序列 (LCS)

## 📌 一、 题目深度剖析

**核心概念区分**：
* **子串 (Substring)**：必须是连续的字符（比如 `ABC` 是 `XABCY` 的子串）。
* **子序列 (Subsequence)**：可以不连续，但必须保持原有的相对顺序（比如 `ABC` 是 `AXBXC` 的子序列）。

**题目目标**：给定两个字符串 $X$ 和 $Y$，找出它们能共有的最长的子序列长度。

---

## 🧠 二、 核心算法思维：从“剪尾巴”到“二维建表”

这道题如果从前往后穷举，会有 $2^N$ 种可能，必定超时。计算机科学中最优雅的解法是**逆向思维（从尾巴开始看）**。

假设我们只看 $X$ 和 $Y$ 的**最后一个字符**，此时只会面临两种命运：

1.  **命运 A：尾巴字符竟然一样！**
    既然一样，那这个字符**绝对**可以作为公共子序列的最后一个字母。我们把它“剪”下来（长度 + 1），然后安心地去比较两个字符串剩下的前面部分。
2.  **命运 B：尾巴字符不一样。**
    既然不一样，它们绝不可能同时出现在公共子序列的末尾。我们需要做两次尝试：
    * **尝试一**：假装 $X$ 的尾巴没用，用 $X$ 剩下的部分去和完整的 $Y$ 比。
    * **尝试二**：假装 $Y$ 的尾巴没用，用完整的 $X$ 去和 $Y$ 剩下的部分比。
    然后，我们在两种尝试中**取最大值**！

为了不重复计算这些剪碎的“小问题”，我们召唤出一个**二维表格**，从长度为 1 的小碎片开始，把答案一路填满到完整长度。

---

## 📝 三、 动态规划状态转移方程

### 1. 状态定义
我们定义一个二维数组 $dp[i][j]$：
* **含义**：表示字符串 $X$ 的**前 $i$ 个字符**与字符串 $Y$ 的**前 $j$ 个字符**的“最长公共子序列长度”。

### 2. 状态转移方程（核心数学翻译）
当我们站在 $dp[i][j]$ 这个格子上准备填数字时，回头看一眼 $X$ 的第 $i$ 个字符（代码中下标为 $i-1$）和 $Y$ 的第 $j$ 个字符（代码中下标为 $j-1$）：

* **如果 $X[i-1] == Y[j-1]$ （字符匹配）**：
    $$dp[i][j] = dp[i-1][j-1] + 1$$
    *(大白话：等于左上角的值加 1)*

* **如果 $X[i-1] \neq Y[j-1]$ （字符不匹配）**：
    $$dp[i][j] = \max(dp[i-1][j], dp[i][j-1])$$
    *(大白话：等于上方格子和左方格子中的最大值)*

### 3. 边界初始化
如果其中一个字符串长度为 0（即前 0 个字符），那根本不可能有公共子序列。
所以，第 0 行和第 0 列全部初始化为 0。

## 📊 四、 DP 状态表推导演示

[交互沙盘模拟](https://lbcpp.readthedocs.io/zh-cn/latest/alg/%E5%8A%A8%E6%80%81%E8%A7%84%E5%88%92.html)

为了直观理解，我们用极简数据 $X = \text{"ABC"}$， $Y = \text{"BDC"}$ 来推演这个填表过程。

| $Y$ \ $X$ | 0 (空) | 1 (`A`) | 2 (`B`) | 3 (`C`) |
| :--- | :--- | :--- | :--- | :--- |
| **0 (空)** | 0 | 0 | 0 | 0 |
| **1 (`B`)** | 0 | 0 (`A!=B`, 取上左最大0) | **1** (`B==B`, 左上0+1) | 1 (`C!=B`, 取上左最大1) |
| **2 (`D`)** | 0 | 0 (`A!=D`) | 1 (`B!=D`, 拿上面的1) | 1 (`C!=D`, 拿上面的1) |
| **3 (`C`)** | 0 | 0 (`A!=C`) | 1 (`B!=C`) | **2** (`C==C`, 左上1+1) |

填表完毕！右下角的 **2** 就是最长公共子序列（`BC`）的长度！

## 💻 五、 满分 C++ 代码实现

代码异常简洁，多开一行一列 `n+1` 和 `m+1` 的目的是为了自然处理 $i-1$ 和 $j-1$ 的越界问题（即第 0 行和第 0 列的初始状态 0）。

```cpp
#include <iostream>
#include <string>
#include <vector>
#include <algorithm>

using namespace std;

int main() {
    string X, Y;
    // 持续读取直到没有输入
    if (cin >> X >> Y) {
        int n = X.length();
        int m = Y.length();

        // 1. 定义 DP 表并初始化为 0
        // dp[i][j] 表示 X 的前 i 个字符和 Y 的前 j 个字符的 LCS 长度
        vector<vector<int>> dp(n + 1, vector<int>(m + 1, 0));

        // 2. 核心状态转移
        for (int i = 1; i <= n; ++i) {
            for (int j = 1; j <= m; ++j) {
                // 注意：字符串下标从 0 开始，所以前 i 个字符的最后一个是 X[i-1]
                if (X[i - 1] == Y[j - 1]) {
                    // 尾巴字符相同：取左上角对角线的值 + 1
                    dp[i][j] = dp[i - 1][j - 1] + 1;
                } else {
                    // 尾巴字符不同：取上方和左方的最大值
                    dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]);
                }
            }
        }

        // 3. 输出右下角的最优解
        cout << dp[n][m] << endl;
    }
    return 0;
}
```

## ⏱️ 六、 复杂度分析
* **时间复杂度**：$O(N \times M)$。双层循环遍历了整个二维数组，每次状态转移只需 $O(1)$ 的时间。本题数据规模 $1000 \times 1000 = 10^6$，在 C++ 1 秒时限内绰绰有余。
* **空间复杂度**：$O(N \times M)$。开辟了一个大小为 $(N+1) \times (M+1)$ 的二维数组。（进阶技巧：由于每次计算当前行只依赖于“当前行”和“上一行”，可以使用**滚动数组**将空间复杂度优化至 $O(M)$，但对于 $N, M \le 1000$ 而言，二维数组完全可以接受）。
