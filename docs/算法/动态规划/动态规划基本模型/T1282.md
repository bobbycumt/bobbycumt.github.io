### T1282最大子矩阵和

[▶ 查看动画](/html/T1282.html)

### 一、考查的语法和技能

1. 二维数组输入与遍历。
2. 双层枚举“上边界 + 下边界”进行矩阵压缩。
3. 一维最大子段和（Kadane 动态规划）。
4. 变量滚动更新与全局最优维护。
5. 时间复杂度分析（$O(n^3)$ 经典做法）。

---

### 二、思路解析

这题是“二维转一维 + 一维 DP”的经典模型。

#### 1）固定上下边界，做列压缩

- 枚举子矩阵的上边界 `top`。
- 再枚举下边界 `bottom`（`bottom >= top`）。
- 在 `top..bottom` 之间，把每一列的元素和累加到 `colSum[c]`。
- 这样，原问题就变成：在数组 `colSum` 中找“最大连续子段和”。

#### 2）对 `colSum` 运行 Kadane

设：
- `cur`：以当前位置结尾的最大和；
- `best`：当前这次压缩下的全局最大和。

转移公式：

$$
cur = \max(colSum[i],\ cur + colSum[i])
$$

并持续更新：

$$
best = \max(best,\ cur)
$$

#### 3）更新总答案

每次 `top-bottom` 压缩完并跑完 Kadane 后，用 `ans = max(ans, best)` 更新最终答案。

### 三、带详细注释的代码（C++）

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    // a[i][j]：矩阵元素（1-based 也可以，这里用 0-based）
    vector<vector<int>> a(n, vector<int>(n));
    for (int i = 0; i < n; ++i) {
        for (int j = 0; j < n; ++j) {
            cin >> a[i][j];
        }
    }

    // 结果可能是负数，所以初值设为极小
    int ans = INT_MIN;

    // 枚举上边界 top
    for (int top = 0; top < n; ++top) {
        // colSum[c] 记录从 top 到 bottom 在第 c 列的累加和
        vector<int> colSum(n, 0);

        // 枚举下边界 bottom
        for (int bottom = top; bottom < n; ++bottom) {
            // 把第 bottom 行加进来，实现二维 -> 一维压缩
            for (int c = 0; c < n; ++c) {
                colSum[c] += a[bottom][c];
            }

            // 对压缩后的一维数组 colSum 做 Kadane
            int cur = colSum[0];
            int best = colSum[0];
            for (int i = 1; i < n; ++i) {
                // 要么从当前单独开始，要么接在前面的最优后面
                cur = max(colSum[i], cur + colSum[i]);
                best = max(best, cur);
            }

            // 更新全局答案
            ans = max(ans, best);
        }
    }

    cout << ans << '\n';
    return 0;
}
```

---

### 四、易错点解析

1. **全负矩阵处理错误**：`ans` 不能初始化为 `0`，应初始化为 `INT_MIN`。
2. **每次新 top 忘记清空 `colSum`**：必须在外层每个 `top` 开始时重新置零。
3. **Kadane 初值写错**：建议从 `colSum[0]` 起步，避免把空子段当合法答案。
4. **复杂度误判**：该写法是三层主循环（`top/bottom/col` + Kadane），总体 $O(n^3)$。
5. **下标混乱**：0-based 与 1-based 混用会导致越界或漏算。
