# 累加和
 [点击链接进行累加和动画互动](/html/累加.html)

**“累加和”**（Accumulator Pattern）是构建程序控制流的基础模型之一，它是学生从“单次计算”迈向“循环迭代计算”的关键跳板。无论是求总分、统计数量，还是后续学习的前缀和算法，都离不开这个核心思想。

### 题目背景：班级总分与平均分统计

**题目描述**
期末考试结束了，班主任需要统计全班同学的数学总分。已知班级共有 N 名学生，请输入这 N 名学生的成绩，并计算出全班的总分。

**输入格式**
第一行包含一个正整数 N（1 <= N <= 100000），表示班级的人数。
第二行包含 N 个非负整数，分别表示每位同学的成绩（单科成绩在 0 到 150 之间），数字之间用一个空格隔开。

**输出格式**
输出一行，包含一个整数，表示全班同学的数学总分。

### 一、 考查知识点与编程技能

1. **累加器思想（Accumulator）**：掌握利用一个变量不断叠加新值的编程模式。
2. **`for` 循环与边读边算**：理解如何在读取数据的同时进行计算，从而优化空间复杂度（空间复杂度从 O(N) 降为 O(1)）。
3. **变量的初始化**：深刻理解局部变量在 C++ 中不自动清零的特性。
4. **数据类型与溢出防范**：学会根据数据范围预估结果大小，选择合适的整型（如 `long long`）。

### 二、 题目思路分析（累加器模型讲解）

**教学引导步骤：**

1. **生活化比喻（引入“存钱罐”概念）**
   * 提问学生：如果现在有 5 个人排队给你交班费，你会怎么算总共收了多少钱？
   * **模拟过程**：你首先需要准备一个**空盒子**（也就是初始值为 0 的存钱罐）。
   * 第 1 个人交了 10 元，你放进盒子里，现在盒子里的钱是：0 + 10 = 10。
   * 第 2 个人交了 15 元，你放进盒子里，现在盒子里的钱是：10 + 15 = 25。
   * 以此类推，每一次操作都是把“新收到的钱”加到“盒子原有的钱”上。

2. **核心代码公式化**
   * 我们把存钱罐命名为 `sum`。每次收到的钱命名为 `score`。
   * 核心动作只有一句：`sum = sum + score;`（也可以简写为 `sum += score;`）。
   * 等号右边的 `sum` 是老状态，加上新数据后，赋值给等号左边的 `sum` 变成新状态。这就叫**状态的迭代更新**。

3. **空间优化：还需要数组吗？**
   * 很多初学者第一反应是开一个大小为 100000 的数组 `a[100000]` 把成绩全存下来，然后再开一个循环遍历求和。
   * **引导拔高**：既然我们只需要知道总和，存钱罐已经把钱装进去了，我们还需要记住每个人具体交了多少钱吗？不需要！所以我们可以**边读入、边累加**，连数组都不用开，极致节省内存。

### 三、 备课代码（附完整注释）

```cpp
#include <iostream>

using namespace std;

int main() {
    // 优化输入输出流
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int n;
    if (!(cin >> n)) return 0;

    // 【步骤 1：准备存钱罐并清零】
    // 极其重要：必须初始化为 0。
    // 因为极限情况下 100000 人，每人 150 分，总和为 15000000，虽然 int 存得下，
    // 但养成好习惯，求和变量遇到数据量大时推荐使用 long long。
    long long sum = 0; 

    // 【步骤 2：排队收钱，边读边加】
    for (int i = 0; i < n; ++i) {
        int score;
        cin >> score; // 读取当前同学的成绩
        
        // 核心累加操作：把新成绩加入存钱罐
        sum += score; 
    }

    // 【步骤 3：输出总和】
    cout << sum << "\n";

    return 0;
}
```

### 四、 易错点与教学提示

在学生上机实操这段看似极其简单的代码时，往往会暴露出最基础但也最致命的 3 个问题：

1. **未初始化累加变量（“脏存钱罐”问题）**
   * *现象*：学生写了 `long long sum;`，然后直接进循环累加。最后输出一个莫名其妙的巨大数字（比如 4235489431）。
   * *纠正*：C++ 里的局部变量如果没有赋初值，它里面装的是内存里残留的“垃圾值”。如果不把存钱罐先清空为 0，就相当于别人往一个原本就装着假钞的盒子里投钱，总数肯定错。**铁律：累加器必须明确初始化为 0。**

2. **累加变量定义在了循环内部（“金鱼记忆”问题）**
   * *现象*：学生把代码写成：
     ```cpp
     for (int i = 0; i < n; ++i) {
         long long sum = 0; // 错误位置
         int score;
         cin >> score;
         sum += score;
     }
     ```
     最后发现求出来的和永远是最后一个人的成绩。
   * *纠正*：这里要强调**变量的生命周期（作用域）**。如果在 `for` 循环内部定义 `sum`，这就好比每来一个同学，你都拿出一个全新的空盒子让他放钱，上一个盒子直接扔掉。所以存钱罐必须放在循环的**外面**。

3. **数据溢出问题（隐形地雷）**
   * *拓展点拨*：本题数据小（1500万），`int`（上限约 21 亿）完全够用。但如果是求 N 个员工的年薪总和，或者题目给的 N 达到 10^5，每个数字达到 10^9，加起来就会爆 `int` 变成负数。
   * *教学建议*：借此机会培养学生的“边界嗅觉”，教导他们：**看到“求和”两个字，条件反射先估算最大可能值，该上 `long long` 绝不犹豫。**
