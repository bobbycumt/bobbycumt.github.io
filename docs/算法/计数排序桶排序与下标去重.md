## 计数排序、桶排序与数组下标去重

[▶ 查看动画](/html/计数排序桶排序下标去重.html)

### 一、考查的语法和技能

1. **数组作为“桶”使用**：把“值”映射到“下标”，例如 `cnt[x]` 表示数字 `x` 出现次数。
2. **循环与条件判断**：
   - 第一次遍历原数组，完成计数。
   - 第二次遍历桶数组，按下标顺序还原结果。
3. **去重思想**：
   - 只要 `cnt[x] > 0`，说明 `x` 至少出现过一次。
   - 输出时每个值只输出一次，实现“数组下标去重”。
4. **复杂度分析能力**：理解为什么计数排序在值域较小时非常快。

---

### 二、思路解析

我们把这题拆成三步：

#### 第 1 步：读入并统计

准备一个桶数组 `cnt`，初值全为 `0`。

遍历输入数组 `a[i]`：

- 执行 `cnt[a[i]]++`，记录每个值出现了多少次。
- 同时维护最大值 `mx`，减少后面遍历桶的范围。

#### 第 2 步：按值从小到大扫描桶

从 `x=0` 到 `mx` 遍历：

- 若 `cnt[x] > 0`，说明这个数字出现过。
- 如果我们要**去重后的升序序列**，就把 `x` 输出一次。

这一步本质就是“用下标当有序键”。因此天然是升序。

#### 第 3 步：对比计数排序和桶排序

- **计数排序**：桶里通常存“次数”，再按次数回填（可保留重复）。
- **桶排序（基础版）**：桶里可存“是否出现/一组元素”，再按桶顺序输出。
- **数组下标去重**：是计数/桶思想的直接应用，只看 `cnt[x] > 0`。

---

### 三、带有详细注释的代码（C++）

> 下面代码演示“去重 + 升序输出”；若要完整计数排序（保留重复），见注释中的扩展写法。

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    vector<int> a(n);

    // 假设数据范围是 [0, 100000]
    // 如果题目范围不同，这个上界要跟着改
    const int MAXV = 100000;
    vector<int> cnt(MAXV + 1, 0);

    int mx = 0; // 记录输入中的最大值，减少桶扫描范围

    for (int i = 0; i < n; ++i) {
        cin >> a[i];

        // 计数：值作为下标
        cnt[a[i]]++;

        // 更新最大值
        if (a[i] > mx) mx = a[i];
    }

    // --- 去重后的升序输出 ---
    // 只要出现过（cnt[x] > 0），就输出一次 x
    bool first = true;
    for (int x = 0; x <= mx; ++x) {
        if (cnt[x] > 0) {
            if (!first) cout << ' ';
            cout << x;
            first = false;
        }
    }
    cout << '\n';

    // --- 扩展：如果要完整计数排序（保留重复）---
    // for (int x = 0; x <= mx; ++x) {
    //     while (cnt[x]--) {
    //         cout << x << ' ';
    //     }
    // }

    return 0;
}
```

### 四、易错点解析

1. **桶数组开小了（越界）**
   - 例如数据有 `100000`，你只开到 `cnt[99999]` 就会越界。
   - 一定先看清值域，再决定桶大小。

2. **有负数时直接当下标**
   - `cnt[-3]` 非法。
   - 处理方式：要么平移下标（加偏移量），要么改用 `map/unordered_map`。

3. **把“去重输出一次”写成“输出 cnt[x] 次”**
   - 这会变成完整计数排序，不是去重。

4. **忘记控制输出空格**
   - 常见问题：行末多空格或开头多空格。
   - 解决：使用 `first` 标记位。

---

### 五、复杂度

- 设输入规模为 `n`，值域上界为 `V`。
- 时间复杂度：`O(n + V)`。
- 空间复杂度：`O(V)`。

当 `V` 不大时，这类方法非常高效；当 `V` 远大于 `n` 时，空间会偏大，要考虑其他结构。
